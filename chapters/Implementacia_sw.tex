\chapter{Implementácia softvérovej časti}
\label{ImplementaciaSW}

Táto kapitola opisuje programové vybavenie, so zreteľom na využitie systémového prístupu a minimalizáciu čistých výpisov kódu v prospech názorných blokových schém.

\section{Firmvér ESP32}
\label{FirmverESP32}

Kód na koncových uzloch je písaný v jazyku C++ s využitím frameworku Arduino.
\begin{itemize}
    \item \textbf{Inicializácia:} 
    \begin{itemize}
        \item Uzol sa prebúdza pripojením na pevnú lokálnu Wi-Fi sieť.
        \item Nasleduje autorizácia a pripojenie na centrálny MQTT Broker.
        \item Publikovanie LWT \textit{online} stavu na diagnostický topik.
    \end{itemize}
    \item \textbf{Vykonávacia slučka (Loop):}
    \begin{itemize}
        \item Prijímanie payload správ odoberaných z topiku (napr. \texttt{SPEED:120}).
        \item Rozparsovanie textového príkazu a jeho mapovanie na fyzickú inštrukciu (nastavenie PWM striedy pre riadiaci pin alebo \texttt{digitalWrite} na piny relé).
        \item Po úspešnom vykonaní odoslanie potvrdzujúcej \texttt{feedback} správy pre odomknutie stavového automatu.
    \end{itemize}
\end{itemize}

\section{Backend Raspberry Pi}
\label{BackendRPI}

Hlavný proces je naprogramovaný v jazyku Python a využíva asynchrónne programovanie.
\begin{itemize}
    \item \textbf{Hlavná trieda (\texttt{MuseumController}):} Koordinuje beh iných vlákien a drží inštanciu MQTT klienta.
    \item \textbf{Inverzia závislostí (\texttt{ServiceContainer}):} 
    \begin{itemize}
        \item Namiesto tvrdého prepájania tried využíva architektúra kontajner služieb.
        \item Pri štarte dynamicky načítava handlery pre audio, video a parser scén, čím zabezpečuje maximálnu softvérovú modularitu zadania.
    \end{itemize}
\end{itemize}

\section{Stavový automat a~scene engine}
\label{StavovyAutomat}

Riadiaca logika expozície nahrádza tradičné procedurálne podmieňovanie konečným automatom.
\begin{itemize}
    \item \textbf{Reprezentácia stavov:} Každá logická časť výstavy je jedným objektom obsahujúcim zásobníky úloh (\texttt{onEnter}, \texttt{timeline}, \texttt{transitions}).
    \item \textbf{Asynchrónny prechod udalosťami:}
    \begin{itemize}
        \item Engine nie je zablokovaný čakaním v slučke (`delay`).
        \item Prechody sú vyhodnocované reaktívne vo forme tzv. \textit{Eventov} (udalostí) prijímaných z frontend dashboardu alebo od senzorických ESP32 uzlov cez MQTT správu.
        \item Ak je podmienka triggra splnená, \texttt{TransitionManager} zruší bežiace asynchrónne časovače v súčasnom stave a prepne automat do stavu nového.
    \end{itemize}
\end{itemize}

\section{Webový dashboard}
\label{WebovyDashboard}

Riadiace centrum pre personál múzea, prístupné z tabletu alebo mobilu vizuálnou formou.
\begin{itemize}
    \item \textbf{Tech stack:} Využitie Python mikroframeworku \textbf{Flask} a asynchrónnej obojsmernej komunikácie cez \textbf{Socket.IO}.
    \item \textbf{Funkcionality operátora:}
    \begin{itemize}
        \item \textbf{Monitoring:} Živé zobrazenie pripojených ESP32 uzlov a aktuálne prebiehajúceho stavu FSM.
        \item \textbf{Manuálne zásahy:} Možnosť interaktívne spustiť/zastaviť scénu (\textit{Master Override}) a núdzovo zablokovať prúdy do motorov.
        \item Čítanie serverových chybových logov prichádzajúcich v reálnom čase zo \texttt{ServiceContainer}-a.
    \end{itemize}
\end{itemize}

\section{Konfigurácia scén -- JSON formát}
\label{KonfiguraciaScen}

Umožňuje editáciu scén bez nutnosti rekompilácie jadra.
\begin{itemize}
    \item \textbf{Formát uzlov:} Koreň s atribútom \texttt{initialState} a poľom \texttt{states}, ktoré obsahuje abstraktné definície choreografie.
    \item \textbf{Polia vnútri stavu:}
    \begin{itemize}
        \item \texttt{onEnter:} Priame pole akcií (napr. mqtt príkazy alebo spustenie zvuku s \texttt{audio: "file.wav"}).
        \item \texttt{timeline:} Pole objektov \texttt{delay} s \texttt{action}.
        \item \textbf{Vetvenie:} Ukážka poľa \texttt{transitions}, kde uzol \texttt{goto} definuje ďalší stav na základe prijatia asynchrónnej zhody typu \texttt{message} od konkrétneho aktéra.
    \end{itemize}
\end{itemize}
